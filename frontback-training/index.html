实验1：把项目文件提交到远程 Git 仓库

【实验名称】

实验1：把项目文件提交到远程 Git 仓库

【实验目的】

  • 联系学习过的 Git 命令，使用 Git 来管理代码。

【实验原理】

  • 注册 Gitee 账号，创建仓库
  • 获取 Git 客户端，通过 Git 访问仓库

【实验环境】

  • 操作系统: Windows系统。
  • Git 客户端 Cmder

【实验资源】

  • Git 客户端 cmder.7z：是一个集成 Git 的命令行的终端软件。

  • 文本编辑器 Notepad++：是一个优秀的文本编辑器。

https://pan.baidu.com/s/1gukX79FA_Lcg2IFFaW_hzA#提取码:eenh

【实验步骤】

  1. 在Gitee.com，登录你的个人账号，创建一个名字为springboot-demo的仓库。

  1. 选择一个本地空目录，例如D:\workspaces\workspace2。打开 cmder，把远程仓库克隆到本地。
cd D:\workspaces\workspace2
git clone <你的仓库SSH协议地址>

  1. 把项目文件全部拷贝进本地仓库目录。

  2. 进入本地仓库目录D:\workspaces\workspace2\springboot-demo，使用以下命令提交文件到远程仓库。这一步也可以使用 IDEA 的 Git 插件提交。

cd D:\workspaces\workspace2\springboot-demo

#把项目文件加入到暂存区
git add *

#提交到本地仓库
git commit -m "初始化项目"

#提交到远程仓库的 master 分支
git push -u origin master
  1. 使用 IDEA 导入本地 Git 仓库的项目。打开菜单File->Open...，选择springboot-demo目录进行导入。

实验2：HTTP GET 和 POST 请求处理

【实验名称】

实验2：HTTP GET 和 POST 请求处理

【实验目的】

  • 掌握 HTTP GET 和 POST 的基本原理和特点，学会编写 Spring Boot 相关代码能够处理 GET 和 POST 请求。

【实验原理】

HTTP request 的 method 主要有 GET 和 POST，GET 通常用于查询操作，POST 通常用于创建、更新和删除资源等操作。

本实验通过模拟登录页面，让学生直观体验 HTTP request 的 GET 和 POST 的区别。

本实验用到的 Thymeleaf（Thyme leaf读作泰丽夫，原意是百里香叶） 模板。模板是一种常见的编写网页的技术。通过往编写好的模板内填充数据，可以获得非常好的网页输出性能，而且相比 JSP 技术拥有更好的灵活性，编写更简便。相似的技术还有 freemaker，velocity等。

【实验环境】

  • 操作系统: Windows系统。
  • IDEA
  • Spring Boot

【实验步骤】

  1. 使用 IDEA 打开 demo 项目。在com.ruiyi.demo.rest路径下新增一个SignInController的类。代码如下：

        }
    }
}

     * 处理异步请求，接受 URLEncoded-Form数据，并返回json数据
     * @param username

    @PostMapping(value = { "/signInWithForm" }, produces= MediaType.APPLICATION_JSON_VALUE)

        ResultVO<String> rvo=new ResultVO<>();
        rvo.setCode(0);//0表示出错
        if("john".equals(username) && "john#2020".equals(pwd)){
            rvo.setCode(1);
            rvo.setMsg("登录成功");
            rvo.setData(username);
        }else{
            rvo.setCode(0);
            rvo.setMsg("登录失败，请检查用户名和密码是否正确！");
            rvo.setData(StringUtils.stripToEmpty(username));
        }
        return JSON.toJSONString(rvo);
    }

    /**
     * 处理异步请求，接受 JSON 数据，并返回json数据
     * @param map
     * @return

    public String signInWithJson(@RequestBody Map<String,Object> map){
        String username=(String)map.get("username");
        String pwd=(String)map.get("pwd");

function buildFormData(){
   var pairs=[],data=["username","pwd"];
   data.forEach(function(name) {
      var input=document.getElementById(name);
      if(input){
        pairs.push(encodeURIComponent(name) + '=' + encodeURIComponent(input.value));
      }
   });

   // 将配对合并为单个字符串，并将所有%编码的空格替换为
   // “+”字符；匹配浏览器表单提交的行为。
   return pairs.join('&').replace(/%20/g, '+');
}

function buildJsonData(){
   var pairs=[],data=["username","pwd"];
   data.forEach(function(name) {
      var input=document.getElementById(name);
      if(input){
        pairs.push('"'+name + '":"' + input.value+'"');
      }
   });
   return "{"+pairs.join(',')+"}";
}

//点击按钮时执行函数
function postForm(){
   //新建一个 XMLHttpRequest 对象
   var request=new XMLHttpRequest();

   //使用 URLEncoded Form 格式发送请求数据。第1个参数GET/POST,第2个是发送的URL，第3个是异步还是同步，true代表异步，不要同步；
   request.open("POST","/signInWithForm",true);
   request.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded');//设定发送数据类型
   request.send(buildFormData());//发送数据

   //使用JSON格式发送请求数据
   //request.open("POST","/signInWithJson",true);
   //request.setRequestHeader('Content-Type', 'application/json');//设定发送数据类型
   //request.send(buildJsonData());//发送数据

   //第四步 监听 request 响应状态
   request.onreadystatechange=function(){
       //readyState 有这几种状态，我们只需要判断4就行;
       // 0: 请求未初始化
       // 1: 服务器连接已建立
       // 2: 请求已接收
       // 3: 请求处理中
       // 4: 请求已完成，且响应已就绪
       if(request.readyState==4){
            //status 状态200 代表 ok
           if(request.status==200){
               //request.responseText代表后台返回的数据
               //JSON.parse可以解析JSON字符串为 javascript 对象
               var rs=JSON.parse(request.responseText);
               alert(rs.msg);
           }
       }
    }
}
