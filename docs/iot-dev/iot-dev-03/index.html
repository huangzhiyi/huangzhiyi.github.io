实训 3.1 管理数据库脚本

【实验名称】

实训 3.1 管理数据库脚本

【实验目的】

  • 掌握数据库脚本的管理

【实验环境】

  • Windows 7 以上64位操作系统

【实验资源】

  • JDK 8 - Java Development Kit是 Oracle 公司针对Java开发人员发布的免费软件开发工具包，是 Java 开发必备的开发工具。
  • Intellij IDEA - 业界简称IDEA，是 jetbrains 公司推出的和 Eclipse 齐名的 Java 集成开发环境（IDE）。
  • Apache Maven - Apache Maven，是一个软件项目管理及自动构建工具，由Apache软件基金会所提供。是 Java 构建打包最广泛使用的工具。
  • MySQL 世界上最受欢迎的开源关系数据库之一。
  • HeidiSQL，开源数据库客户端。
  • Cmder 是一个美化的 Windows 命令行终端，支持 ctrl-V 粘贴。
  • Notepad++ 是一个开源文本编辑器，支持代码着色等多种功能，比 Windows 原生文本编辑器功能要丰富很多。
  • Flyway 是一个管理数据库脚本迁移的工具。
  • Eclipse Mosquitto 是一个开源的 MQTT 服务器。支持5.0、3.1.1和3.1的MQTT协议。
  • MQTT.fx 是一个基于 JavaFX 的 MQTT 客户端.

【实验步骤】

  1. 启动MySQL。参考Part1。

  2. 打开 HeidiSQL 连接 MySQL。参考Part1。

  3. 在客户端执行以下脚本，创建一个名为iotweb的数据库。并且创建名为iotweb的用户，密码为iot123。

create database iotweb;
create user 'iotweb'@'localhost' identified by 'iot123';
create user 'iotweb'@'%' identified by 'iot123';
grant all on iotweb.* to 'iotweb'@'localhost';
grant all on iotweb.* to 'iotweb'@'%';

  1. 使用 HeidiSQL 使用iotweb用户连接数据库iotweb。测试是否能够正常连接。

  2. 使用 IDEA 打开iotweb这个项目。在pom.xml 的build标签内加入以下代码。新增 Flyway 用于管理数据库 SQL 脚本。

	<build>
		<plugins>
			<plugin>
				<groupId>org.springframework.boot</groupId>
				<artifactId>spring-boot-maven-plugin</artifactId>
			</plugin>
			<!-- 新增 flyway 管理数据库脚本-->
			<plugin>
				<groupId>org.flywaydb</groupId>
				<artifactId>flyway-maven-plugin</artifactId>
				<version>7.8.2</version>
				<configuration>
					<url>jdbc:mysql://localhost:3306/iotweb?useUnicode=true&amp;characterEncoding=UTF-8&amp;useSSL=false</url>
					<user>iotweb</user>
					<password>iot123</password>
				</configuration>
			</plugin>
			<!--  flyway 结束-->
		</plugins>
	</build>
  1. 在src\main\resources下创建2级目录用于存放 SQL 脚本。如下所示 
src
  |-main
    |-resources
	  |-db
	    |-migration
  1. 在src\main\resources\db\migration下创建一个名称为V001__CREATE_EMOJI.sql的文件（注意文件名有2个连续下划线），并输入以下内容。
CREATE TABLE `emoji` (
	`id` BIGINT(20) UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键',
	`name` VARCHAR(256) NOT NULL COMMENT '名称',
	`face` VARCHAR(100) NOT NULL COMMENT '表情',
	PRIMARY KEY (`id`) USING BTREE
)
COMMENT='emoji表情'
COLLATE='utf8_general_ci'
ENGINE=InnoDB
ROW_FORMAT=DYNAMIC;
  1. 在 IDEA 的 Terminal 下运行命令，开始递增式执行脚本。
mvn flyway:migrate

  1. 使用 HeidiSQL 查看iotweb数据库有什么不同，并记录到课程设计报告。

  2. 在src\main\resources\db\migration下创建一个名称为V002__INSERT_EMOJI.sql的文件（注意文件名有2个连续下划线），并输入以下脚本。

INSERT INTO `emoji` (`name`,`face`) values('sad','╭(╯^╰)╮'),('smile','(*￣︶￣)');
  1. 在 IDEA 的 Terminal 下运行命令，开始递增式执行脚本。
mvn flyway:migrate
  1. 使用 HeidiSQL 查看iotweb数据库有什么不同，并记录到课程设计报告。

【实验步骤】

  1. 在iotweb项目中新建1个DBTests的单元测试用例，代码如下。
import org.junit.jupiter.api.Test;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.ResultSet;
import java.sql.SQLException;
import java.sql.Statement;

public class DBTests {

    static final String JDBC_DRIVER = "com.mysql.cj.jdbc.Driver";
    static final String DB_URL = "jdbc:mysql://localhost:3306/iotweb";
    static final String USER = "iotweb";
    static final String PASS = "iot123";

    @Test
    public void test_select() {
        Connection conn = null;
        Statement stmt = null;
        try {
            // 注册 JDBC 驱动
            Class.forName(JDBC_DRIVER);

            // 打开链接
            conn = DriverManager.getConnection(DB_URL, USER, PASS);

            // 执行查询
            stmt = conn.createStatement();
            String sql;
            sql = "SELECT name, face FROM emoji";
            ResultSet rs = stmt.executeQuery(sql);

            String name = null;
            String face = null;
            // 展开结果集数据库
            while (rs.next()) {
                // 通过字段检索
                name = rs.getString("name");
                face = rs.getString("face");

                // 输出数据
                System.out.println("name:" + name + " face:" + face);
            }
            // 完成后关闭
            rs.close();
            stmt.close();
            conn.close();
        } catch (SQLException se) {
            // 处理 JDBC 错误
            se.printStackTrace();
        } catch (Exception e) {
            // 处理 Class.forName 错误
            e.printStackTrace();
        } finally {
            // 关闭资源
            try {
                if (stmt != null) stmt.close();
            } catch (SQLException se2) {
                se2.printStackTrace();
            }
            try {
                if (conn != null) conn.close();
            } catch (SQLException se) {
                se.printStackTrace();
            }
        }
    }
}

  • EmojiController
package com.example.iotweb.controller;

import com.example.iotweb.service.EmojiService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.servlet.ModelAndView;

@Controller
public class EmojiController {

    @Autowired
    private EmojiService emojiService;

    @GetMapping("/getEmojiByName")
    public String getEmojiByName(@RequestParam(name = "name") String name,Model  model){
        model.addAttribute("emoji",emojiService.getFaceByName(name));
        return "emoji";
    }
}

package com.example.iotweb.repos;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.jdbc.core.JdbcTemplate;
import org.springframework.jdbc.core.SqlTypeValue;
import org.springframework.stereotype.Repository;

public class EmojiRepositoryImpl implements EmojiRepository{
